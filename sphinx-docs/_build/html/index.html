static from_file(path: str, target='bug', col_start=3, col_stop=-2, hooks: list | None = None) → Data¶

Path to file

Parameters:

  • path – Path to file

  • target – Target column

  • col_start – Column to start reading at

  • col_stop – Column to stop reading at

  • hooks – List of hooks. This is passed after the columns are filtered. The data is passed as a DataFrame (x) and a Series (y), before splitting for train/test.

Returns:

Data object

static from_files(base_path: str, files: list, target: str = 'bug', col_start: int = 3, col_stop: int = -2, n_classes: int = 2, hooks: list | None = None) → Data¶

Builds data from a list of files, the last of which is the test set.

Parameters:

  • base_path – The path to fetch from

  • files – List of files. The last one is the test set.

  • target – Target column

  • col_start – Column to start reading from. 3 for PROMISE defect prediction.

  • col_stop – Column to stop reading. -2 for PROMISE defect prediction.

  • n_classes – Number of classes.

  • hooks – List of hooks. These are passed the train/test DataFrames after filtering columns

Returns:

Data object

raise_utils.hooks¶

class raise_utils.hooks.Hook(name: str, function: Callable)¶

The base class for hooks. User-defined hooks must be a derived class of this class. Hooks can be used in various places to introduce user-defined behavior in addition to the standard operations performed by the package.
